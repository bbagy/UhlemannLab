Merge & Rename

ONT sequencing output often contains many barcodeXX/ directories, each with many .fastq.gz files. The merge/rename step combines files barcode-by-barcode and then renames them using a mapping table.

Supported Map Formats

old  new
barcode01  KP0011
barcode02  KP0063

Headerless old new format is also supported, and legacy sample barcode files remain supported.

Standard Merge

bash Go_merge_rename.sh -i fastq_pass -o merged_fastqs -m rename_map.tsv -n
bash Go_merge_rename.sh -i fastq_pass -o merged_fastqs -m rename_map.tsv

Size-Limited Merge

Go_merge_rename_V1_0.sh adds --merge-until-size so you can stop merging once the cumulative compressed size reaches a practical threshold such as 200M, 500M, or 1G.
  • Files are merged whole-file at a time.
  • No .fastq.gz file is ever cut in the middle.
  • If the size threshold is not reached, all available files are merged for that barcode.
bash Go_merge_rename_V1_0.sh -i fastq_pass -o merged_fastqs -m rename_map.tsv -n --merge-until-size 200M
bash Go_merge_rename_V1_0.sh -i fastq_pass -o merged_fastqs -m rename_map.tsv --merge-until-size 500M

Quick Start

./longWGS/Go_longWGS_V1_1.sh \
  -i /path/to/fastq \
  -o /path/to/output \
  -d /path/to/db \
  -M strict \
  -p 0 \
  -K

Real Example

Go_longWGS.sh \
  -i 1_merged_fastqs \
  -o 2_longWGS_out \
  -d /media/uhlemann/Core3_V2/DB/longWGS_DB \
  -s /home/uhlemann/heekuk_path \
  -M strict \
  -p 0 \
  -K

Options

FlagDefaultDescription
-i-Input FASTQ directory
-o-Output directory
-d-DB root mounted to container as /db
-sscript directoryOptional Snakefile directory override
-p00: skip porechop, 1: run porechop
-MstrictPipeline mode: strict stops on stage failures, permissive records failures and still builds available summaries
-noffDry-run
-KoffKeep-going on independent failures

Common Runs

# dry-run
./longWGS/Go_longWGS_V1_1.sh -i IN -o OUT -d DB -n

# production
./longWGS/Go_longWGS_V1_1.sh -i IN -o OUT -d DB -M strict -K

# enable porechop
./longWGS/Go_longWGS_V1_1.sh -i IN -o OUT -d DB -p 1 -M strict -K

# continue to summary workbook even if some Bakta jobs fail
./longWGS/Go_longWGS_V1_1.sh -i IN -o OUT -d DB -M permissive -K

Permissive Mode

Use -M permissive when you want the run to finish and still write checkm2_coverage_summary.xlsx even if some Bakta jobs fail late in the pipeline.
  • Failed Bakta samples are recorded in 0_failed_samples.tsv.
  • 7_bakta/<sample>/FAILED.txt marks samples where Bakta failed.
  • In the summary workbook, RunStatus uses bakta_done for true success and bakta_failed for skipped-after-failure samples.

Troubleshooting

  • docker: image not found longwgs: build with docker build -t longwgs longWGS
  • Kraken2 DB not found under DB root: place a Kraken2 DB under DB/kraken2DB/<your_db> or related supported layouts
  • lock-related failure in log: wrapper already retries with --unlock
  • repeated prefilter not running: expected when input files are older than 0_bad_fastqs/DONE.txt
  • no Bandage images: install Bandage on host or skip
  • Bakta failures block the final Excel: rerun with -M permissive -K
