Kraken2 / Bracken

KBracken

Kraken2 + Bracken profiling with merged MPA-style outputs and master logs.

STATUS: IN TESTING

Workstation Layout

heekuk_path/
|-- Go_KBracken.sh
|-- Go_KBracken.smk
|-- scripts/
|   |-- bracken_to_mpa.py
|   |-- fill_bracken_taxonomy.py
|   |-- kraken_masterlog.py
|   `-- merge_mpa_tables.py
`-- docker/KBracken/
    `-- Dockerfile

Build

cd /home/uhlemann/heekuk_path/docker/KBracken
docker build --network=host -t kbracken:1.0 .

Run

Go_KBracken.sh \
  -i /path/to/fastq_dir \
  -o KBracken_out \
  -d /path/to/kraken2_db \
  -c 8 \
  -j 4 \
  -K

Input Naming

ASB_1_R1.fastq.gz / ASB_1_R2.fastq.gz -> sample ASB_1
ASB_2_R1.fastq.gz / ASB_2_R2.fastq.gz -> sample ASB_2
S1_1.fastq.gz and S1_2.fastq.gz stay separate single-end samples

Paired-end inference requires explicit R1/R2 or forward/reverse tokens.
