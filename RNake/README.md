# RNake — Bacterial RNA-seq pipeline (Snakemake)

RNake is a Snakemake pipeline for **bacterial RNA-seq analysis**.  
It is designed for convenience and reproducibility, supporting both **single-end** and **paired-end** FASTQ inputs.

---

## What this pipeline does

- Read trimming  
- Mapping to a bacterial reference genome  
- Read counting (gene-level)
- Merging per-sample counts into a final count table

---

## Requirements

- Snakemake
- Conda (recommended)  
- Reference genome FASTA (`.fna` / `.fa`)
- Reference annotation GFF (`.gff`)

Tools used (installed via conda envs):
- Trimmomatic
- Bowtie2
- SAMtools
- HTSeq

---

## Quick start

Example command:

```bash
pro="your_project_name"
fastqs="your_fastq_directory"
genome="your_reference_genome.fna"
gff="your_reference_annotation.gff"

snakemake \
  -s /your_path/Go_bacteriaRNake_paired_V3.smk \
  --cores 16 \
  --config project="$pro" read_dir="$fastqs" genome="$genome" gff="$gff" \
  --rerun-incomplete \
  --use-conda
```

---

## Inputs

### FASTQ files
- Supports both **single-end** and **paired-end**
- Expected naming patterns depend on the Snakefile version  
  (recommendation: keep consistent Illumina style `*_R1_*.fastq.gz`, `*_R2_*.fastq.gz`)

### Reference genome
- FASTA format (`.fna`, `.fa`)

### Reference annotation
- GFF format (`.gff`)

---

## Prokka GFF note (important)

If your GFF was generated by **Prokka**, it often includes a FASTA section at the end.  
This can break downstream parsing.

Remove the FASTA part using:

```bash
sed '/^##FASTA$/,$d' your_prokka_output.gff > your_cleaned_output.gff
```

---

## Output

The main output file is:

- `merged_counts_with_gene_names.csv`

This file contains:
- merged gene counts across all samples  
- gene names extracted from the provided GFF annotation  

It is located in the project output directory.
